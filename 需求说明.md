# 需求说明 · B站影视灵感雷达可视化

本文件记录本项目页面的原始需求与确认口径，作为后续维护与迭代的依据。

## 一、原始需求

依据 `B站影视灵感雷达` 目录下每日扫描沉淀的数据，生成一个 HTML 页面并部署到仓库 `https://github.com/archili2035/film_radar`，具体要求：

1. **index 页面需在仓库根目录**。
2. 数据源是分天扫描的，但为了一次浏览与筛选，**在一个页面里统一汇总统计**；唯一区别是不同条目可标注各自的扫描时间。
3. 视觉呈现上需支持**筛选、排序**等功能。

## 二、确认口径（决策记录）

| 决策项 | 选择 | 说明 |
| --- | --- | --- |
| 数据粒度 | 以作品为主行 | 每张卡片为一个作品，视频作为卡片内子列表展示 |
| 低置信/噪声样本 | 纳入但打标签 | 保留在数据中，标注置信度，默认通过开关隐藏 |
| 部署方式 | 通过 GitHub 更新仓库 | 目标为根目录 `index.html` + GitHub Pages |
| 视觉风格 | 暗色数据雷达风 | 深色背景、雷达扫描氛围、数据卡片式布局 |

## 三、功能范围

- **聚合展示**：多天日报合并为单页；每个条目标注 `scanDate`。
- **搜索**：覆盖作品名、UP主、视频标题、BV号、灵感关键词。
- **排序**：播放量、互动量、点赞、收藏、投币、内容信号、扫描日期、名称。
- **筛选**：扫描日期、题材线、置信度。
- **作品卡片**：展示 B站信号、内容信号、主要讨论点、可迁移灵感方向、后续动作，以及可折叠的相关视频列表与详细热度数值。
- **顶部概览**：标题区、关键统计卡片、各次扫描的时间线标签。

## 四、数据与边界

- 数据来源为 B 站公开搜索接口（WBI 签名）的定时扫描快照。
- 热度数值为扫描当时的公开统计，会随时间变化；**B站热度不等同作品质量或参考优先级**。
- 同一作品在不同扫描日出现时，拆分为多个条目（不同 `id` + `scanDate`），保留按扫描时间标注的原始信息，不做跨天合并。
- 本页面仅用于灵感信号的汇总与检索，不做内容评价或参考优先级结论。

## 五、实现与部署要点

- `index.html` 为单文件自包含实现：数据内嵌为 `const DATA`，无外部脚本/接口依赖。
- `data.json` 为同一份数据的独立可维护副本。
- 部署经 git 推送到 `main` 分支根目录，并开启 GitHub Pages（`main` / 根目录）。
  - 注：GitHub 只读集成连接器无法写入本仓库（403），实际采用本地 git + 已授权凭据推送，效果与"通过 GitHub 更新仓库"一致。

## 六、后续可迭代方向

- 增量更新流程：新增扫描日后追加 `data.json` 并回填 `index.html` 内嵌数据。
- 可考虑改为运行时 `fetch('data.json')`，让页面与数据解耦（会牺牲纯离线双击可用性，需权衡）。
- 可增加题材线维度的统计图或趋势对比。
